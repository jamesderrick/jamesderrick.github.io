WordPress Deployment Governance

  • Why: A level of development control was required for this client. Dependant on third party developers it was hard to manage multiple updates while making certain the sites would experience minimal downtime.
  • What: With a combination of tried and tested plugins allowing the sites to adopt a 3 phase process, and a semi-automated deployment process at minimal extra cost to the client being built within the previously designed architecture, this allowed all developers to work in an environment that would not effect production before pushing changes to a staging environment for review before the client could then deploy changes to production when satisfied.
  • Extra Details: This project involved a lot of face to face discussions with the client to bottom out a solution that would meet their needs perfectly.
