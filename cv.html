Class

WordPress Migration

  • Why: After the successful recreation of the training site within WordPress, and it's successful running within AWS it was decided to move their main corporate website to AWS.
  • What: A much simpler project, given that the original site was already in WordPress, all that was required was moving the database to an RDS and replicating the filesystem within an AWS compute engine instance. To cut down on maintenance the 2 sites were merged into a WordPress multisite, which also brought with it it's own issues around access levels. A custom hierarchy was put in place to support the client's needs.
  • Extra Details: To improve the usability of the site for developers an automated deployment process was added. This used AWS CodeCommit as the trigger and a small Talend service to authenticate and trigger the git pull process. This allowed their third party developer to easily make changes to a staging site before pushing code into production, speeding up their deployment process and minimising risk.

Custom Web Application

  • What: Utilising Servian's UI/UX team to design the front end screens, it was my job to create the website that the customers would use.
